Cobot Magic Fine-Tuning and Benchmark Methodology
=================================================

Audit date: 2026-07-09.

This document covers only Being-H0.5, SmolVLA, GR00T N1.7, and GreenVLA.
OpenVLA is intentionally excluded.

The methodology was reconstructed from README files, training scripts, source code,
training logs, checkpoint configs, and checkpoint metadata. When README commands and
checkpoint metadata disagree, the checkpoint metadata is treated as the source of
truth.


1. Dataset
----------

Main dataset:
/home/dual4090/workspace/apanasevich/cobot_magic_sber

Format: LeRobot v2.1.
Size: 7341 episodes, 609146 frames, 1858 task descriptions, 15 FPS.

Camera streams:
- camera_0: right wrist camera;
- camera_1: left wrist camera;
- camera_2: high/front camera.

Raw `observation.state` and `action` are 26-dimensional:
- indices 0:14 contain the joint states for both arms;
- indices 14:26 contain FK/end-effector xyz/rpy values.

For VLA fine-tuning, the models used the joint-only 14D part of the state/action
space. In the dataset, `action[t]` semantically corresponds to
`observation.state[t + 1]`, with the final frame patched by repeating the final
available state. Most fine-tuning runs converted actions into relative joint deltas
with respect to the current state.

Dataset QA:
- OK episodes: 7338;
- excluded short episodes: 1609, 4221, 5650.

3. Being-H0.5

Training hardware:

- checkpoint step: 50000;

- learning_rate: 1e-4;

7. Benchmark
------------

Result directory:
/home/dual4090/workspace/apanasevich/benchmark_results_summer2026

Aggregation/plotting script:
/home/dual4090/workspace/apanasevich/benchmark_aloha/realworld_exp/plot_results.py

Benchmark files:
- 4 models: Being-H05, SmolVLA, GrootN1.7, GreenVLA;
- 2 task families: objects and colors;
- 2 hands: left_hand and right_hand;
- 16 JSON result files in total.

Each JSON file contains:
- experiment_name;
- policy_name;
- task_config_path;
- seed: 20260429;
- num_sequences: 5;
- num_tasks: 5;
- total_trials: 25;
- trial_plan;
- trials with the fields `instruction_understood` and `task_success`.

Protocol:
- each model x task_family x hand combination was evaluated on 25 trials;
- 25 trials = 5 sequences x 5 tasks;
- `source_num_attempts` in the JSON is ignored; the benchmark uses the saved
  `trial_plan` from the task config;
- aggregation counts only trials that contain both `instruction_understood` and
  `task_success`;
- metrics:
  1. Instruction understanding = fraction of trials with `instruction_understood=True`;
  2. Task success = fraction of trials with `task_success=True`.

Objects tasks:
- pliers;
- sponge;
- screwdriver;
- battery;
- tape.

Colors tasks:
- white object;
- yellow object;
- blue object;
- green object;
- red object.

All instructions followed the template "Pick up ... and place it in the plastic
container with the left/right hand". The benchmark therefore measured both language
grounding for object/color selection and successful pick-and-place execution with
the requested arm.

- results/4models_averaged_hands.svg: left/right hands averaged by task family;
- results/4models_left_hand.svg: left hand only;
